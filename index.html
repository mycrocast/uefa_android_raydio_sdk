RaydioSDK

The RaydioSDK will enable you to easily integrate listening to live-audio broadcasts with low-latency, which were created by the mycrocast platform.

The following documentation will guide you through the setup and usage of the Android RaydioSDK for UEFA in more detail.

1. Initial Setup

Add the following maven-repository to your dependencyResolutionManagement in your settings.gradle file:

maven("https://mycrocast-maven-repo.s3.eu-central-1.amazonaws.com/android")

After this the RaydioSDK can resolve every dependencies it internally has and you can add the artifact of the RaydioSDK to your module dependencies: 

dependencies {
	...
    
    implementation("de.mycrocast.uefa.raydio:1.0.0")
    
    ...
}

Now you can integrate the RaydioSDK into your application!

2. Integration of the SDK

The first thing you need to to use the SDK in your application is to create an instance of the RaydioSDK as this class contains all classes you need to integrate the functionality of the SDK.

Use the RaydioSDKBuilder to create an instance of RaydioSDK. Hold the reference to this created instance for the whole lifetime of your application because you should only create it once and reuse it.

To create the RaydioSDKBuilder you need to create an implementation of the RaydioSDKCredentials interface and an instance of SharedPreferences. The last one is only used to store a unique String-identifier of the user of the RaydioSDK and therefore your application, as this allows us to identify reccurring users.

INFORMATION The generation of the user identifier is completely randomized in the backend of Raydio and does not include any critical or sensitive information about the user, their mobile phone nor your application.

The RaydioSDKCredentials is needed to identify your application and to match the live-audio broadcasts of your streamers, which were started in the www.studio.mycrocast.de. So you will only get informed about livestreams with the same clubId as the on you set in the credentials.

INFORMATION You will get your clubId after the registration from us.

I would strongly advise to use a dependency injection library for the integration of the RaydioSDK, e.g. with Dagger-Hilt like in the example project. But it is also possible to setup the sdk in your custom application class or in a completely seperate singleton, if you want. But make sure you only create an instance once and reuse it.

3. Important interfaces

Before describing how to use the sdk and its interfaces in more detail, lets take a look to core interface you will stumble on.

RaydioLivestream

The RaydioLivestream interface represents a live audio broadcast which was started by a streamer in the www.studio.mycrocast.de. It holds every information about a single audio broadcast you will need, like the title and language of the broadcast or the matchId used to identify which match this livestream belongs to. (For more detail see the documentation)

Language

The Language interface holds information about the language of a RaydioLivestream which was configured be the streamer in the creation of a broadcast. The selected language is represented in the interface in three different ways: The english name of the language, the code of the language (according to the ISO 639-1) and the native representation. The last one is how the language itself is called in the specific language (e.g. 'Deutsch' for German or 'Fran√ßais' for French).

RaydioLivestreamGroup

According to the specification you gave us, you wanted to group the livestreams by their title. For this the RaydioLivestreamGroup interface was created. One instance of it holds all RaydioLivestream's with the same title AND the same matchId and gives you easy accessability to both properties.

INFORMATION The list of RaydioLivestream's of a RaydioLivestreamGroup will NEVER be empty.

3. Usage of the SDK

After the setup and the configuration of the SDK in the previous chapter, this one describes the intended way of using the RaydioSDK. For the following steps you need to use the interfaces given to you via the RaydioSDK.

The first thing you need to do is to use the RaydioConnection to connect to the backend of Raydio. If this succeeds you should load currently active RaydioLivestreams via the RaydioLivestreamLoader. This will change the list of active RaydioLivestreamGroups in the RaydioLivestreamGroupContainer. If a user selects a RaydioLivestream of a RaydioLivestreamGroup to play, you should start a Foreground Service to play the audio broadcast of this livestream via the RaydioLivestreamPlayer, which can be created with the RaydioLivestreamPlayer.Factory.

Establishing a connection and observing changes

For establishing a connection as well as observing the connection state use RaydioConnection. At the start before the first connection attempt the initial value of currentState will be NEW. This is the perfect time to try establishing a connection to the Raydio backend via connect. You will need an active network connection for this to succeed. While trying to establishing a connection, the currentState changes to CONNECTING. Here you could show a loading animation, if you want. Depending on whether the connection attempt succeeds or fails the currentState changes either to CONNECTED for success or DISCONNECTED for failure.

If the connection attempt fails, you will get informed by the reason if you attached a RaydioLogger to the RaydioSDKBuilder while building the RaydioSDK. (see [#4. Logging]). Of course you can try to establish a connection again either via connect or reconnect.

If the connectin attempt succeeds, you can load the currently active RaydioLivestreams as described below.

If you established a connection once it is possible that this connection is lost after some time. This is exactle the case if the currentState changes from CONNECTED to DISCONNECTED. Here you need to implement a reconnect use case (again via connect or reconnect) according to your specifications. I would advise you to reconnect on demand (e.g. the user has a reconnect button) if no livestream is currently playing and if one is playing i would try to reconnect in a timely manner (see the example application).

INFROMATION While the SDK is not connected, the following requests will fail:

  • RaydioLivestreamLoader: load

  • RaydioLivestreamPlayer: play

If a request fails, the SDK will log it as an error in your provided RaydioLogger.

Loading and observing RaydioLivestreamGroup changes

After establishing a connection to the Raydio backend you can request all currently active live broadcasts via the RaydioLivestreamLoader. Therefore just call the function load. If this succeeds, meaning if it returns true, it will update the property online containing the list of currently active RaydioLivestreamGroups in the RaydioLivestreamGroupContainer. In the beginning, before the first loading, this property contains an empty list. If the process fails, returning false, no changes in the RaydioLivestreamGroupContainer will be made.

If a RaydioLivestream and therefore a RaydioLivestreamGroup was added (new livestream was started in the platform), changed (streamer adjusted the livestream) or removed (streamer stopped streaming) the list of the online property will be adjusted automatically and you will get informed about changes if you collect this flow.

INFORMATION If the connection to the Raydio backend was lost and after some time reconnected, it is necessary that you reload the list of currently active livestreams, as this could have changed in the time the client was disconnected.

To get updates of your currently playing RaydioLivestream you can collect the flow of the find(streamerId: String) method, where the parameter streamerId is the Identifier of the streamer of the corresponding RaydioLivestream. You should do this in your Foreground Service, which is playing the RaydioLivestream. This flow will contain the value null exactly when no RaydioLivestream with the given streamerId was found. If you collect the flow in the ForegroundService and the value is null, the streamer either stopped the audio broadcast of the RaydioLivestream or has problems with his internet connection (triggered by a timeout of the corresponding RaydioLivestream in the backend). Here you could stop the service immediately or wait a specific time, because the streamer can reconnect to his audio broadcast. If the streamer reconnects, the value of the flow will be the reconnected RaydioLivestream, which has a different identifier than the previous one. This means you need to call 'play()' of the RaydioLivestreamPlayer again with the identifier of the new RaydioLivestream to continue the play.

Playing a RaydioLivestream

For playing a RaydioLivestream you need to create a RaydioLivestreamPlayer via the RaydioLivestreamPlayer.Factory.

After the creation of a RaydioLivestreamPlayer the initial value of current, alias the current PlayState is New. If you want to start playing a RaydioLivestream call the play method with the identifier of the RaydioLivestream. This will change current to Connecting and the player tries to establish a connection to the audio broadcast of this RaydioLivestream. If this succeeds current will change to Playing, if it fails it will change to Failed.

If a connection was established previously (meaning current holds the value Playing) and the connection was lost (due to connectivity issues on client or streamer side), current changes to Disconnected and after some time it will change to Failed. Each of these PlayStates holds the identifier alias id of the RaydioLivestream.

If you want to stop the playing of a RaydioLivestream invoke the function stop of the RaydioLivestreamPlayer you used to start the playing process. This will also stop an ongoing connecting process (current is still Connecting and not yet Playing). After stopping is done current will be Closed.

INFORMATION First i would strongly advise to only use the RaydioLivestreamPlayer in a Foreground service, as his functionality was only tested in one and can be only guaranteed in one.

Also i would recommend to create a new RaydioLivestreamPlayer via a new Factory for each Foreground service you start and not reuse old ones as this can lead to unwanted and unpredictable PlayState changes.

For a good example how to use a Foreground service with the RaydioLivestreamPlayer, which also handles connection changes of client and streamer side, look into the LivestreamPlayService of the example application

4. Logging

While creating an instance of the RaydioSDK via the RaydioSDKBuilder you have the possibility to add a RaydioLogger to the sdk. This will give you additional information about the processes in the sdk itself as well as the option to log interactions of the users. These interactions are of the type RaydioInteraction. An interaction is logged whenever you start or stop the playing of a livestream with the information about which is the current user and what is the current livestream (both via their unique identifier).

Otherwise the sdk logs information, warnings and errors. While the information logs are just for understanding and tracking the process of the SDK, warnings may guide you to potential pitfalls and errors will occur only when an exception was thrown inside the SDK. Even if an exception was thrown the SDK itself will not crash.

INFORMATION It will help us a lot if you inform us if you encounter an error and how to reproduce it.

5. How to test? How to create a broadcast?

Login

Visit the mycrocast platform and in the upper tab select "Streamer" and login with the credentials provided.

Start stream

Hit the stream start button and:

  • Select a genre

  • Select a language

  • Select Event

  • Enter a title

  • Select a microphone

  • Hit continue -> you are live a few moments later

INFORMATION The livestream creation process for UEFA will be simplified in the near future.

Use audio file as content

Instead of your own voice you can use an audio file which will be broadcasted.

To do so:

  • Drag and drop a file into the upper right corner of the website

  • Hit the play button on the entry that appeared there

How to create multiple entries in the same group

You need to create multiple streams, you need to log in with different users in different tabs. At the moment each stream must have the EXACT same title!

INFORMATION A future update from our backend and webplatform will change the livestream starting process for UEFA where the streamer just needs to select the language in which he wants to broadcast in and the game he wants to broadcast. (No RaydioSDK update is needed for this!)

Current shortcomings

MatchId of RaydioLivestreamGroup and RaydioLivestream

Currently the provided matchId is always just "0". This will change in the future with an update of our webplatform and backend. (No RaydioSDK update is needed for this!)

RaydioLivestreamGroup creation

As previously described multiple RaydioLivestreams are grouped into one RaydioLivestreamGroup exactle when their title matches. Therefore if you want to create a RaydioLivestreamGroup you currently need to create multiple livestreams with the exact same title.
